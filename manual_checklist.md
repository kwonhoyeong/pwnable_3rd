# Final System Health Check - Manual Checklist

이 문서는 `pwnable_3rd` 시스템의 최종 배포 전, 자동화 스크립트로 확인하기 어려운 UI/UX 및 브라우저 동작을 점검하기 위한 체크리스트입니다.

## 1. 웹사이트 직접 점검 (Direct Website Check)
- [ ] **브라우저 접속**: `http://localhost:3000`에 접속하여 페이지가 정상적으로 로딩되는지 확인합니다.
- [ ] **콘솔 로그 확인**: 개발자 도구(F12) > Console 탭을 열어 붉은색 에러 로그(Error)가 없는지 확인합니다. (경고(Warning)는 허용 가능)
- [ ] **화면 깨짐 확인**: 다양한 해상도(데스크탑, 태블릿, 모바일)에서 레이아웃이 깨지지 않는지 확인합니다.

## 2. "Dark Sentinel" 디자인 테마 점검
- [ ] **컬러 팔레트**: 배경색이 Deep Charcoal(`#111315`)이고, 포인트 컬러가 Royal Purple(`#6C5DD3`)인지 확인합니다.
- [ ] **폰트 적용**: 모든 텍스트에 `Inter` 폰트가 적용되어 깔끔하게 표시되는지 확인합니다.
- [ ] **Insight Cards**: 대시보드 상단의 스탯 카드가 단순 박스가 아닌, 아이콘과 그라데이션 효과가 적용된 디자인인지 확인합니다.
- [ ] **헤더 삭제**: 상단 헤더(검색창, 알림 등)가 삭제되고, 사이드바와 메인 콘텐츠만 표시되는지 확인합니다.

## 3. 기능 동작 점검
- [ ] **패키지 검색**: 대시보드에서 패키지명을 입력하고 검색했을 때, 로딩 인디케이터가 표시되고 결과가 정상적으로 나타나는지 확인합니다.
- [ ] **에러 처리**: 존재하지 않는 패키지를 검색하거나 네트워크를 끊고 검색했을 때, 적절한 에러 메시지(Toast 또는 배너)가 표시되는지 확인합니다.
- [ ] **상세 리포트**: 검색 결과의 상세 페이지에서 마크다운 콘텐츠(취약점 설명 등)가 올바르게 렌더링되는지 확인합니다.
- [ ] **히스토리 페이지**: 사이드바의 '히스토리' 메뉴로 이동하여 이전 검색 기록이 표시되는지, 한글화가 잘 되어 있는지 확인합니다.

## 4. 데이터 및 Fallback 점검
- [ ] **AI 데이터 Fallback**: 백엔드에서 AI API 키가 비활성화되어 있을 때, "AI 분석 불가" 등의 메시지와 함께 기본 데이터가 표시되는지 확인합니다.
- [ ] **데이터 갱신**: 검색 후 새로고침 시 데이터가 유지되거나 빠르게 로딩되는지(캐싱 동작 체감) 확인합니다.
